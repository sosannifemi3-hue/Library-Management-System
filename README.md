# Library-Management-System
1. Requirement Analysis
This phase defines what the Library Management System should do.
• Functional Requirements
- The system shall accept book title and author name
- The system shall store book records
- The system shall display available books
• Non-Functional Requirements
- The system should be easy to use
- The system should be reliable
2. System Design
•This phase describes how the system will be developed.
•Programming Language: Python
•Interface: Command Line Interface
•Data Storage: List
•System Components
Book
Library Management System
3. Implementation
•The Library Management System was implemented using Python classes and lists to store book records.
4. Testing
•The system was tested by:
Adding book records
Displaying stored books
The system produced the expected output.
5. Deployment
•The project was deployed by:
Running the Python program locally
Uploading the source code to GitHub
6. Maintenance
•Maintenance includes fixing bugs and adding new features such as deleting or updating book records.
